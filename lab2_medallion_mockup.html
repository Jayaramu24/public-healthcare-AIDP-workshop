Task 9

Create the AIDP workflow for incremental medallion runs

10 minutes
Instructor demo with participant review

After participants have run the notebooks manually, show the validated AIDP Workflow that turns those notebooks into an operational pipeline. This section intentionally comes after the manual notebook work so first-time participants understand the layers before seeing orchestration.

Task 9 step 1

Open Workflow and create the job

In AIDP Workbench, open the shared workshop workspace, click Workflow, and create a workflow named MPHA_INCREMENTAL_MEDALLION_FLOW.

Task 9 step 2

Configure the Bronze ingest task

Rename the first task to bronzeingest. Select the task type that matches the uploaded asset. In the validated run, Bronze used a Python task pointing to /Workspace/O1_Bronze/aidp_bronze_pyspark.py.

  • Select the Bronze file from the O1_Bronze folder.
  • Select compute E2EAIDPIndustrydemos.
  • Set timeout to 60 minutes before adding the next task.
Task 9 step 3

Add Silver after the timeout is set

Add silverrefine only after the Bronze task has a timeout value. Set the dependency to bronzeingest, choose the Silver notebook, select the same compute, and set timeout to 60 minutes.

Task 9 step 4

Add Gold staging and AI Lakehouse load tasks

Add goldstage after silverrefine. Then add lakehouseload after goldstage. For both tasks, select Notebook task, choose the file from the correct workspace folder, select compute, and set timeout to 60 minutes.

Task 9 step 5

Review and run the workflow

Before running, confirm the chain is bronzeingest to silverrefine to goldstage to lakehouseload. Run the workflow and watch the graph view.

Task 9 step 6

Confirm task progress and final success

Open task details as the workflow runs. The Bronze output proves the raw files were written to the shared volume. The final lakehouseload task confirms the Claims star schema load.

Task 9 troubleshooting

Use Repair run for a failed downstream task

If Bronze, Silver, and Gold already succeeded, do not rerun the full chain. Fix the notebook issue, click Repair run, select only lakehouseload, and run the repair.

bronzeingestPreserve new raw files and stamp the batch.
silverrefineClean and standardize impacted records.
goldstageStage broader Gold outputs.
lakehouseloadInsert only new Claims star schema rows into AI Lakehouse.
ConceptWorkshop explanationProduction incrementality
Batch idUse a readable value such as mpha_2025_07_01_daily.Labels every run and supports replay, audit, and duplicate detection.
WatermarkUse watermark_start and watermark_end to describe the new data window.Filters raw files or records so only new or changed data is processed.
TimeoutEvery workflow task uses 60 minutes in the validated run.Prevents long-running jobs from hanging indefinitely.
Gold loadThe Claims star schema load reads existing target keys and writes only new rows.Production logic can evolve into true dimension upserts and fact-grain refreshes.
Repair runIf only lakehouseload fails, fix the notebook and repair just that task.Reduces rerun cost while preserving successful upstream lineage.
